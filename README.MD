## рҹ“қ Reglas de IngenierГӯa y Ciclo de Vida (Para el Equipo)

Antes de modificar el cГіdigo, el equipo de desarrollo debe comprender las directrices obligatorias del proyecto para no corromper la arquitectura:

1.  ****Sin Frameworks Reactivos:**** El frontend se rige por la manipulaciГіn directa de nodos del DOM (`document.createElement`).
2.  ****Limpieza Estricta de Memoria:**** Antes de montar una nueva vista en el orquestador (`main.js`), el contenedor principal (`#app`) se limpia obligatoriamente con `appElement.innerHTML = ''`. Esto destruye los nodos anteriores y previene fugas de memoria o duplicaciГіn de escuchadores de eventos (`EventListeners`).
3.  ****Enrutamiento por Hash:**** No se realizan peticiones de recarga al servidor para cambiar de pГЎgina. La SPA escucha el evento `hashchange` del navegador (`#/`, `#/tutor`, `#/citas`) y renderiza el componente modular correspondiente en tiempo real.
4.  ****Persistencia Real:**** Se prohГӯbe el uso de almacenamiento volГЎtil del navegador para el flujo transaccional; el agendamiento de citas interactГәa de forma directa con la base de datos SQLite. El `localStorage` se reserva Гәnica y exclusivamente para mantener el estado de la sesiГіn simulada.

## рҹ”„ Flujo Operativo del Proyecto

### \[A\] Flujo del Estudiante (Rol: ESTUDIANTE)

1.  El usuario ingresa a la plataforma y visualiza el catГЎlogo de tutores disponibles.
2.  Puede filtrar la lista interactivamente mediante un selector de lenguajes (`JavaScript`, `Python`, `SQL`).
3.  Al hacer clic en "Ver Perfil Completo", la SPA cambia su hash a `#/tutor?id=X`, cargando los datos especГӯficos del docente, su especialidad y el bloque de comentarios/calificaciones dejados por otros alumnos.
4.  Al pulsar "Agendar Cita", se le redirige al formulario de reserva (`#/citas?tutor=X`) con el docente preseleccionado.
5.  Llena sus datos y la cita se guarda en SQLite, actualizando inmediatamente la tabla de "Historial de Clases Solicitadas".

### \[B\] Flujo del Docente (Rol: TUTOR)

1.  El usuario cambia su cuenta en la barra de navegaciГіn al rol de ****TUTOR (DOCENTE)****.
2.  Al dirigirse al "Panel Citas", la SPA detecta el cambio de estado en el `localStorage`.
3.  El formulario de inscripciГіn desaparece de forma automГЎtica del DOM para proteger el flujo.
4.  La interfaz muta y renderiza una tabla de control exclusiva: ****"Clases que debes Dictar"****, mostrando de forma directa los nombres de los estudiantes aspirantes y los bloques horarios reservados.
